Errors

Something API uses conventional HTTP response codes to indicate the success or failure of an API request. In general: Codes in the 2xx range indicate success. Codes in the 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted etc.). Codes in the 5xx range indicate an error with Something API's servers (these are rare, however if you encounter these, open a ticket in our Discord). Some 4xx errors that could be handled programmatically include an error code that briefly explains the error reported.

HTTP Code Description
200 - OK Everything worked as expected.
400 - Bad Request The request was unacceptable, often due to missing a required parameter.
401 - Unauthorized No valid token provided.
402 - Request Failed The parameters were valid but the request failed.
404 - Not Found The requested resource doesn't exist.
429 - Too Many Requests Too many requests hit the API too quickly.
500, 502, 503, 504 - Server Errors Something went wrong on our end.

Common errors

  1. {"detail":"Method x not allowed."}

    We only allow GET requests.

  2. {"detail":"invalid x content type."}

    Your x query is not in the expected format. Check the documentation for the specific endpoint you want to access.

  3. {"detail":"missing x query."}

    You don't include the required x query in your request.

  4. {"detail":"invalid x query."}

    Your x query is invalid. Check the documentation for the specific endpoint you want to access.

  5. My error is not listed here, what do I do?

    Please create a ticket on our Discord for us to solve the issue.
